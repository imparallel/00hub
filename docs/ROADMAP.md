# 00Hub Future Roadmap & Known Issues

평행 님의 00Hub 프로젝트가 나아갈 방향과 현재 해결해야 할 과제들을 정리한 문서입니다.

## ✅ 최근 해결된 과제 (v1.6.2 완료 — 2026.03.22)

- [x] **시스템 종료 시 taskkill.exe 오류 수정**: 컴퓨터 종료 시 런처가 서버를 종료하려고 시도하다 발생하는 충돌을 해결했습니다. 시스템 종료 신호(`WindowsShutDown`) 감지 시 명시적 종료 로직을 건너뛰고, 빌드 환경 호환성을 위해 `System.Net.Http` 참조를 추가했습니다.

## ✅ 최근 해결된 과제 (v1.6.1 완료 — 2026.03.22)

- [x] **아카이브 레이아웃 버그 수정**: 보관함 항목이 많을 때 `max-height: 60vh` 제한으로 인해 리스트가 잘리던 현상을 해결하여, Active 탭과 동일하게 자연스러운 확장이 가능하도록 교정했습니다.

- [x] **SplashScreen 안정화 (v1.6.0)**: HMR, SW, sessionStorage 충돌을 해결하여 완벽하게 한 번만 재생되도록 구현
- [x] **런처 로딩 개선**: 고정 딜레이(2초)를 제거하고 서버 응답(HTTP 200) 대기 로직으로 변경
- [x] **문서 무결성 전수 검사**: README.md 및 docs 폴더 내 모든 파일(5종)의 버전, 기능 설명, 코드 구조(누락 19건+) 완벽 동기화

## ✅ 최근 해결된 과제 (v1.5.5 완료)

- **미니멀리즘 익스트림 (Minimalism Extreme)**: 뻔히 보이는 중복 알림(백색소음/타이머)을 제거하고, 토스트 알림을 1.5초로 가속 및 Black 아이콘으로 통일하여 시각적 질서 확립.
- **젠 모드 '무(無)'의 상태 구현**: 젠 모드 상단에서 불필요한 날짜 정보를 제거하고 현재 시간만 노출되도록 개선.

## ✅ 최근 해결된 과제 (v1.5.0 완료)

- **데이터 무결성 강화 (High-Integrity ID)**: UUID 기반의 문자열 ID 체계(`timestamp-random`)를 도입하여 데이터 합치기나 백업 시 발생할 수 있는 ID 충돌 가능성을 원천 차단.
- **원자적 성취도 동기화 (Atomic Heatmap Sync)**: 파편화된 계산 로직을 `useEffect` 엔진으로 통합하여 성능을 최적화하고, 오늘 실시간 수치를 히트맵에 즉각 반영.
- **지능형 과거 데이터 복구 (Legacy Recovery)**: 로우 데이터가 유실된 과거 히트맵 항목이라도 보관소(Archive)와 할 일 목록을 역스캔하여 정확한 수치를 시각화하는 지능형 엔진 탑재.
- **서브태스크 드래그 버그 수정**: 새로운 문자열 ID 체계 도입 후 발생한 세부 할 일 정렬 오류를 완전 해결.
- **포커스 스틸(Focus Stealing) 및 드래그 애니메이션 파손 해결 (Usage Patch)**: WebView2 환경의 강제 포커싱에 대응하는 더미 우회(Dummy Focus Sink) 로직을 도입하여 디원띵 영역의 줌 버그를 차단하고, 보관함(Archive) 드래그 앤 드롭 시 외부 CSS로 인해 발생하는 이중 점프 현상을 완벽히 방어.

## ✅ 최근 해결된 과제 (v1.4.1 완료)

- **강제 상시 최상단 고정 (Forced Always on Top)**: 집중력 유지를 위해 앱을 상시 최상단에 고정하고 Alt+Tab에 의한 화면 가림을 원천 차단.
- **전역 단축키 토글 (Alt + \`)**: 한 손 조작이 편리한 백틱 키 조합으로 앱을 즉시 켜고 끌 수 있는 토글 기능 구현.
- **Zen 모드 몰입 가이드**: Zen 모드 진입 시 단축키(`Alt + \``)를 차단하여 의도치 않은 몰입 이탈을 방지.
- **데이터 무결성 및 수출입 고도화**: 수출입 시 보관함(Archive) 데이터 누락 문제 해결 및 가져오기 시 ID 기반 중복 제거 로직 적용.
- **자정 동기화 및 자동 보관 최적화**: 자정 변경 시 즉각적인 데이터 보관 처리(`todayString` 의존성 강화) 및 보관 이동 시 중복 방지 필터링.
- **AI 유지보수 가이드 구축**: 미래의 에이전트를 위한 빌드 명령어 및 프로젝트 철학을 담은 `AI_MAINTENANCE.md` 작성.

## ✅ 최근 해결된 과제 (v1.4.0 완료)

- **Action Items 슈퍼 보관함**: 날짜별 그룹화 토글, 보관함 항목 수정/삭제, 날짜 내 D&D 및 세부 할 일 토글 기능으로 과거 기록 관리 고도화.
- **UI 대통합 및 시각적 균형**: 모든 위젯 헤더 구조 통합(`card-header-group`) 및 수직 중앙 정렬 최적화, 위젯별 폰트/밑줄 미세 조정.
- **Focus Timer 스마트 인디케이터**: 작동 상태를 직관적으로 알리는 Cyan 점멸 도트 및 중앙 정렬 레이아웃 적용.

## ✅ 최근 해결된 과제 (v1.3.13 완료)

- **런처 지능형 모니터 관리**: 실시간 디스플레이 변경 감지(`SystemEvents`) 및 Win32 API 기반 **모니터 실명(모델명) 표시** 기능 구현.
- **아이콘 안정성 강화**: 빌드 시 원본 PNG로부터 아이콘을 직접 생성하여 아이콘 유실 및 런처 브랜딩 이슈 완벽 해결.

## ✅ 최근 해결된 과제 (v1.1 완료)

- **V1.2 히트맵 칵테일 혁신 및 목표 상향**: 동적 레이어링 구현, 브랜드 컬러 정렬(Y-M-C) 확립, 일일 집중 목표 4시간 상향 및 기존 데이터 자동 보정(0.75x) 반영.
- **V1.1 UX 혁신 및 용어 정립**: 프리미엄 알림 시스템(Toast/Modal), 서브태스크 상세 제어(D&D/편집/용어 리팩토링), 과거 기록 보호 로직 완료.

- **데이터 안정성**: 기존 데이터를 유실 없이 새 구조(Subtask)로 전환하는 자동 마이그레이션 적용.
- **문서 및 설정 최신화**: v1.1 릴리즈에 맞춰 모든 기술 문서 동기화 및 미사용 레거시 코드 정리 완료.

## ✅ 최근 해결된 과제 (v1.0 완료)

- **상시 거주형 UI**: 작업 표시줄 공간 점유 문제를 시스템 트레이(Tray) 전환으로 완벽 해결.
- **멀티 모니터 대응**: 트레이 메뉴를 통한 모니터 이동 기능을 구현하여 다중 모니터 환경 최적화 완료.
- **몰입형 젠 모드**: 젠 모드 진입 시에만 창을 최상단에 고정하는 동적 TopMost 로직 구현.
- **실행 안정성**: 시작 프로그램 등록 시 경로 공백 처리 및 브랜딩(`00Hub.exe`) 통일.
- **문서 및 설정 최신화**: v1.0 릴리즈에 맞춰 모든 기술 문서 동기화 및 미사용 레거시(Electron) 코드 정리 완료.

## 📝 현재 기술적 부채 및 남아있는 이슈

- **창 모드 레이아웃 불안정성**: 창 크기가 급격히 변하거나(Resize) 전환 시 타이밍 차이로 인한 텍스트 잘림 현상 (상시 전체화면으로 우회 중).
- **데이터 로컬 종속**: 현재 브라우저 내장 저장소(LocalStorage)에만 의존하여 기기 간 동기화 불가능.

## 🚀 향후 고도화 계획 (Next Steps)

### 1. 플랫폼 확장

- [ ] **macOS 버전 개발**: WebView2 대신 macOS의 WebKit(WKWebView) 기반 런처 개발하여 맥 사용자 지원.
- [ ] **모바일/태블릿 앱**: 네이티브 앱(React Native 또는 Capacitor) 환경으로 패키징하여 진정한 모바일 생산성 도구로 확장.

### 2. 기능 고도화

- [ ] **레이아웃 엔진 재설계**: 어떤 창 크기에서도 완벽한 비율을 유지하는 캔버스 기반 렌더링 또는 더 견고한 CSS Grid 시스템 도입.
- [ ] **데이터 동기화**: 현재 LocalStorage 기반인 데이터를 클라우드(Firebase 등)와 연동하여 기기 간 동기화 지원.
- [ ] **다크/라이트 모드 자동 전환**: 시스템 설정에 따른 테마 전환 최적화.

## 🛠 중장기 비전

00Hub를 단순한 대시보드를 넘어, 사용자의 모든 기기에서 동일한 몰입감(Zen)을 제공하는 통합 생산성 에코시스템으로 발전시킵니다.
